## Project Overview
Transformed a provided case study into a sound database design by:
1. Creating a **conceptual model** (ER diagram) based on case study text.
2. Normalising supplied forms and data structures.
3. Integrating conceptual and normalised elements into a **logical model**.
4. Implementing the logical model as an Oracle database schema.

## Key Deliverables
- Conceptual model (oo_conceptual.pdf)
- Normalisation process and results (oo_normalisation.pdf)
- Logical model (oo_logical.pdf)
- Assumptions and design rationale (oo_assumption.pdf)
- Oracle schema creation script (oo_schema.sql)
- Model artifacts archive (oo_model.zip)
- Sample schema execution output (oo_schema_output.txt)

## Technologies
- Oracle RDBMS
- ER/Conceptual Modeling
